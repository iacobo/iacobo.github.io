Stack

I work mainly in machine learning research and so my tech stack centres around Python and PyTorch. Python is the first language I learned way back when and still my go-to for personal and work projects.

I used the open source IDE VSCodium for many years but if you like to tinker you can build your own custom lean IDE with NeoVim.

I've been using SQL for database work for donkey's years. It's a very useful skill to have, most systems use it for databases at some level. I don't have any strong opinions on flavours for personal projects, but in a professional environment you'll likely be exposed to t-SQL.

I've been using LaTeX for typesetting in my mathematical and scientific work since 2012. Recently I discovered Typst, and it's a joy to work with in comparison. Highly recommend anyone frustrated with Latex import conflicts and slow compile times and obtuse error messages give it a try.

I've used various linux distros over the years, but mostly Windows since working after uni. I got tired of all the hacks one has to make to get rid of spyware and ads and telemetry baked into so many components and switched back to linux on my personal devices. Everything is so much cleaner and simpler. I like Gnome, and Debian and Fedora work great on my personal machines, but your (hardware dependent) mileage may vary.

Typing

If you type daily as part of your job you're probably pretty efficient at it. But being born after the 70's I never learned to touch type in school and I found that, while fast, I was having a number of hand and wrist complaints typing hours a day.

This isn't medical advice, it's just a quick overview of the trip I took moving from typing with only my index fingers and stretching to hit shortcuts to touch typing with minimal strain on an ergonomic keyboard.

  • Pick a new layout. I don't think this matters too much per se on an ergonomic board (the point of them is that all keys should be easy to reach, no?), but I chose a new layout since I was learning from scratch and didn't want my index-finger-typing muscle memory to interfere, or be disrupted itself for the times where I'd need to use another person's keyboard. If you like alternations and Hemmingway try Dvorak; if you prefer rolls and Murakami why not Colmak.
  • learn to touchtype: ditchqwerty.com
  • change to an ergonomic keyboard. What's ergonomic for you will depend on your own hands and fingers, and how many rows you are comfortable mentally remapping to layers. There is a lovely open source community of keyboard designs for all preferences. Check them out, see if you see something you like. Below is a favourite: reddit.com/r/ErgoMechKeyboards
